# /api/notes 500 Fix (PR4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **This plan is diagnose-then-fix:** Tasks 1-4 reproduce the failure and identify which hypothesis branch (H1-H4) applies, Task 5+ implements the appropriate fix.

**Goal:** Identify and fix the root cause of `GET /api/notes` returning 500 INTERNAL_ERROR (with a valid Auth0 Bearer) on a freshly-deployed notes-demo. End state: `/api/notes` returns 200 with an empty array on a fresh deploy; create/list/delete round-trip works.

**Architecture:** Diagnostic phase first — deploy from current `main` (PR1+PR2+PR3 already merged), reproduce 500 via Playwright + valid Bearer, read `app` container logs, identify hypothesis branch. Fix phase implements the localized patch and adds a regression test. The plan branches at Task 5 by which hypothesis matched.

**Tech Stack:** Auth0 Management API, Playwright (already used in `apps/cli/test/e2e/`), Dokploy MCP for app discovery, SSH for log reading (`mcp__dokploy__application-readLogs` is unreliable per PR3 README), Vitest, runtime+QSM internals.

**Spec reference:** `docs/superpowers/specs/2026-05-04-notes-demo-fresh-tenant-deployable-design.md` §7.

**Hypotheses (from spec §7.1):**
- **H1**: PDM projection table not bootstrapped → `no such table: notes` SQLITE_ERROR. **Primary hypothesis.** Note: ad-hoc reading shows `bootstrapProjections` IS called for ephemeral mode (sets `seen_events` table), but it iterates `service.projectionDdls`. If the QSM artifact doesn't yield a DDL for the `notes` projection, the table isn't created. Investigation likely lands here.
- **H2**: SQLite DSN/connection error in ephemeral path → `database is locked` / connection refused.
- **H3**: IntrospectSession pre-block returned non-zero gRPC status; bindings runtime mistranslated to INTERNAL_ERROR instead of UNAUTHENTICATED.
- **H4**: bindings-http transport bug crashing on Session serialization.

**File map (Tasks 5+ vary by hypothesis):**
- H1 fix: `packages/runtime/projection-consumer/src/store/bootstrap.ts` and/or `packages/artifacts/qsm/src/` (where ProjectionDdlSpec is built); test in `packages/runtime/projection-consumer/test/`.
- H2 fix: `packages/runtime/runtime/src/plugins/better-sqlite-driver.ts`; test alongside.
- H3 fix: `packages/runtime/bindings-http/` or `packages/runtime/runtime/src/plugins/executors/`; test alongside.
- H4 fix: `packages/runtime/bindings-http/`; test alongside.
- Common: `packages/runtime/runtime/README.md` (lifecycle docs if H1), `demo/notes-blueprint/README.md` (User test after deploy update), e2e gated test addition.

- [ ] **Step 2: Run a Playwright login script to capture an access token**

Create `scripts/probe-notes-demo.mjs` (gitignored or scratch):

```js
import { chromium } from 'playwright';
const url = 'https://notes-demo.rntme.com/';
const email = process.env.PROBE_EMAIL;
const password = process.env.PROBE_PASSWORD;

const browser = await chromium.launch({ headless: true });
const ctx = await browser.newContext();
const page = await ctx.newPage();

let capturedToken = null;
page.on('response', async (res) => {
  if (res.url().includes('/oauth/token') && res.status() === 200) {
    const body = await res.json().catch(() => null);
    if (body?.access_token) capturedToken = body.access_token;
  }
});

await page.goto(url);
await page.click('text=Sign in');
await page.fill('input[name="username"]', email);
await page.fill('input[name="password"]', password);
await page.click('button[type=submit]');
// Auth0 may show a consent screen on first login.
const consent = page.locator('button:has-text("Accept")');
if (await consent.isVisible({ timeout: 5000 }).catch(() => false)) {
  await consent.click();
}
await page.waitForURL(/notes-demo\.rntme\.com\//, { timeout: 30000 });
await page.waitForTimeout(2000); // let token exchange settle

if (!capturedToken) throw new Error('no access_token captured');
console.log(capturedToken);
await browser.close();
```

- [ ] **Step 2: Understand the gap**

- (a) The QSM artifact lists the `notes` projection but the compiler skips DDL emission for SQLite ephemeral mode.
- (b) The blueprint's `qsm.json` doesn't declare a `notes` projection at all (then `listNotes` query routes elsewhere).
- (c) The mapping pipeline runs but the produced DDL has `CREATE TABLE` without `IF NOT EXISTS` and the table genuinely doesn't exist on first deploy → but `bootstrapProjections` already adds `IF NOT EXISTS` (see `packages/runtime/projection-consumer/src/store/bootstrap.ts:6`). So this isn't a gap.

Decide: which of (a)/(b) is true.
